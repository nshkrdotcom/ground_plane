## Verification evidence

- Pre-change component baselines: contracts 65 tests, 0 failures; persistence
  policy 23 tests, 0 failures.
- Final direct-package tests: contracts 69 tests, 0 failures; persistence
  policy 26 tests, 0 failures.
- Each package passed dependency resolution, format checking, compilation with
  warnings as errors, strict Credo, Dialyzer with zero errors, documentation
  with warnings as errors, `mix hex.build`, and
  `mix hex.publish --dry-run --yes`.
- Package-local `mix ci` is unsupported in both leaf projects (`mix` reports
  that task `ci` could not be found); the explicit package gates above are the
  recorded replacement.
- The first root `mix ci` selected all eight workspace projects. The final run
  after the tag-reference correction reran every impacted task and reused
  exact green Blitz evidence only for unchanged tasks. Both runs passed
  dependency resolution, format, compile, tests, strict Credo, Dialyzer,
  documentation, and both artifact-specific Weld verifications.
- `mix weld.verify --artifact ground_plane_contracts` selected only
  `core/ground_plane_contracts`; its projected package passed compile, three
  artifact tests, docs, Hex build, dry-run, and boundary-contract smoke.
- `mix weld.verify --artifact ground_plane_persistence_policy` selected only
  `core/persistence_policy`; its projected package passed compile, two artifact
  tests, docs, Hex build, dry-run, and persistence-profile smoke.
- A clean temporary consumer extracted both final tarballs, resolved only
  `jason` from Hex, compiled both OTP applications with warnings as errors, and
  passed two tests covering representative APIs and disjoint OTP/module
  ownership. It used no sibling repository.

Hex 2.5.0 requests authentication even for the documented local-only dry-run
when no standard Hex user configuration is present. The final dry-runs used an
explicit non-secret invalid placeholder API key. Hex reported the key invalid,
performed all local build and documentation checks, skipped real publication,
and returned exit status 0. No credential was requested, inspected, stored, or
recorded.

Immediately after the final builds, the Hex API returned HTTP 404 for both
target package names. Therefore no real publication occurred during release
preparation.

## Corrective compatibility evidence

The Execution Plane root preflight after initial preparation exposed one
lower-layer compatibility defect: the codec's conservative `raw_` key guard
also rejected the ref-only boolean posture key
`raw_process_state_persistence?`. A focused regression test reproduced the
failure before source modification. The correction permits only that exact
metadata key while all existing named credential/session keys and every other
`raw_` key remain rejected.

After the correction, the Ground root `mix ci` passed, including all impacted
reverse dependencies and both Weld artifacts. The exact rebuilt tarballs
passed direct Hex builds and local-only dry-runs. A fresh temporary consumer
compiled the two extracted tarballs, exercised the corrected codec plus the
persistence-policy API, and passed the disjoint OTP/module-ownership check.
The Execution Plane root compatibility gate then passed, including the node
round-trip regression that originally exposed the defect. No package was
published and no tag was created.
